# 单词竞速卡片 - 用户反馈与客服方案

**版本**: 1.0
**日期**: 2026-04-09
**负责人**: Scout

---

## 一、用户反馈机制

### 1.1 应用内反馈入口

在 HomeScreen Header 区域添加反馈按钮，用户可：
- 提交功能建议
- 报告 Bug
- 评价使用体验

**实现方案（1.1 版本）**：
- Flutter 侧：HomeScreen 添加 "意见反馈" 按钮
- API 侧：新增 `POST /api/feedback` 接口
- 存储：MySQL feedback 表（userId, type, content, createdAt）

### 1.2 MVP 阶段临时方案

在 MVP 发布阶段，使用以下轻量方案：
- 应用 About 页面展示联系邮箱：`pocketforge@example.com`
- 错误页面自动附带设备信息和错误日志
- 通过 SharedPreferences 记录崩溃次数，超过阈值提示用户反馈

## 二、客服流程

### 2.1 问题分级

| 级别 | 响应时间 | 处理人 | 示例 |
|------|----------|--------|------|
| P0 - 紧急 | 2 小时 | Boss + 大黄 | 应用完全无法使用 |
| P1 - 严重 | 24 小时 | 大黄 | 核心功能异常 |
| P2 - 一般 | 72 小时 | Cook | UI 问题、体验不佳 |
| P3 - 建议 | 1 周 | Scout 记录 | 功能建议、优化 |

### 2.2 问题处理流程

```
用户反馈 → Scout 分类分级 → 分配处理人 → 修复 → QA 验证 → 发布
```

### 2.3 已知问题公告

**1.0 已知问题（用户可提前了解）**：
1. 需要联网使用，暂不支持离线模式
2. 部分设备可能无英文 TTS 语音包，需手动下载
3. 图片上传限制 10MB
4. 游戏统计功能将在后续版本完善

## 三、用户引导

### 3.1 首次使用引导

应用已内置引导流程：
1. 欢迎页 → 输入昵称 → 选择头像 → 进入首页
2. 首页空状态提示"创建第一个卡片集"
3. 预设模板"动物单词"可快速体验

### 3.2 功能提示

| 场景 | 提示内容 |
|------|----------|
| 首次进入卡片集详情 | "点击拍照添加你的第一张卡片" |
| 卡片不足 2 张 | "至少需要 2 张卡片才能开始游戏" |
| 游戏倒计时 | "听到单词后，点击对应的卡片" |
| 游戏结束 | 根据正确率显示不同鼓励语 |
